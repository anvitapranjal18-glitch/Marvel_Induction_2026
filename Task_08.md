## **Task 8: Technical Writing & Resource Documentation**

### **Description**

The core objective of this task was to master **Markdown**, a lightweight markup language, to create high-quality, device-agnostic technical documentation. Unlike traditional word processors, Markdown allows for structured formatting—such as hierarchical headings, syntax-highlighted code blocks, and data tables—using plain text. This task involved researching a complex technical subject and distilling it into a "Resource Article" designed for the **MARVEL** website, emphasizing clarity, accessibility, and standardized web formatting.

### **Detailed Process**

- **Topic Selection & Research:** I selected **"The Lifecycle of a Network Request"** as the subject matter. This topic requires explaining the transition from a human-readable URL to a machine-level IP address and subsequent data exchange, making it ideal for structured documentation.
- **Structural Drafting:** I applied a logical hierarchy to the article using ATX-style headers (`#` through `###`). This ensures the document is easily navigable and SEO-friendly.
- **Implementing Advanced Markdown Features:** To elevate the article from a simple text file to a technical resource, I integrated:
  - **Code Blocks:** To demonstrate server-side responses and terminal commands (using backticks).
  - **Tables:** To compare protocols like HTTP/1.1 vs. HTTP/2.
  - **Hyperlinks & Media:** To point readers toward official RFC documentation and external visual aids.
- **Standardization Check:** I verified the document's consistency across different Markdown flavors (GitHub Flavored Markdown vs. Standard) to ensure the article renders perfectly on the MARVEL web portal regardless of the device or browser used.
- **Publishing Workflow:** The finalized `.md` file was processed and uploaded to the community resource repository, following the contribution guidelines provided in the task brief.

### **Featured Resource Article (Sample Excerpt)**

> #### **Understanding DNS Resolution**
